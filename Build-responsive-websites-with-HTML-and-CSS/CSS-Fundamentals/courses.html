Introduction to CSS

A selector in CSS is a pattern that defines which HTML elements on a web page will be styled by a set of CSS rules.

  • Selectors target specific elements based on various criteria, such as element type, class, ID, or their relationship with other elements.
  • CSS selectors play a crucial role in styling and formatting web pages by allowing developers to apply styles selectively to different parts of the HTML document.
  • CSS provides a variety of selectors, including element selectors, class selectors (denoted by a period .), ID selectors (denoted by a hash #), and more advanced selectors like attribute selectors and pseudo-classes.
  • These selectors enable precise targeting of elements for styling purposes.
  • Understanding and using CSS selectors is fundamental for effective web development and design, allowing developers to create visually appealing and well-structured interfaces

is the CSS Rule?

  • A CSS rule is a combination of a selector and a declaration block that defines how a specific set of HTML elements should be styled.
  • The selector indicates which HTML elements the rule applies to, and the declaration block contains one or more declarations, each consisting of a CSS property and its corresponding value.
  • These declarations specify the visual presentation of the selected elements, such as color, size, or positioning.

Inline, Internal and Extrnal

What is Inline, Internal, and External CSS?

  • Inline CSS:
    • Definition: Inline CSS is applied directly to a single HTML element, using the style attribute.
    • Scope: It affects only the specific element it is applied to.
  • Internal CSS:
    • Definition: Internal CSS is defined within the HTML document using the style tag in the head section.
    • Scope: It applies styles to elements on the same page and is not limited to a single element
  • External CSS:
    • Definition: External CSS is stored in a separate file and linked to the HTML document using the tag
    • Scope: It can be applied across multiple pages, promoting a consistent style for the entire website

What is Separation of Concerns?

Separation of Concerns (SoC) is a fundamental design principle in computer science and software development. It involves breaking a computer program or system into distinct and independent sections, with each section addressing a specific concern or aspect of functionality. The goal is to organize and manage complexity by isolating different aspects of the software, making it easier to understand, maintain, and scale

Key points about Separation of Concerns include:

  • Distinct Sections: The principle advocates for separating the codebase into well-defined sections, each handling specific responsibilities.
  • Preventing High Coupling: SoC helps prevent high coupling between different components, allowing changes in one section without affecting others
  • Modularity: It promotes modularity, making it easier to develop, test, and maintain individual modules or components
  • Readability and Maintainability: By organizing code based on concerns, it enhances code readability and makes maintenance more straightforward.

What Does “Don’t Repeat Yourself” Mean in Programming?

Don’t Repeat Yourself” (DRY) is a software development principle that encourages developers to avoid duplicating code. The core idea is to write code in a way that eliminates redundancy and promotes code reuse

What is Semantic Naming in Programming?

Semantic naming in programming refers to the practice of choosing names for variables, functions, classes, or other entities in a codebase that convey meaning about their purpose or functionality. The goal is to use names that provide clear and understandable information about the role and intent of the code element.

  • Descriptive Names: Semantic names are descriptive and reflect the purpose of the entity they represent.
  • Avoiding Ambiguity: Names should be unambiguous, reducing the likelihood of misinterpretation.
  • Conveying Intent: The name itself should convey the intent or behavior of the associated code element.
  • Consistency: Maintaining consistency in naming conventions across the codebase enhances overall clarity.

Semantic naming is a fundamental principle in writing clean and understandable code. It allows developers to quickly grasp the functionality of different parts of the codebase, making it easier to maintain and collaborate on projects

What Values Can the Font Size Take in CSS?

  • Absolute Lengths: Numeric values followed by units like px (pixels), pt (points), cm (centimeters), etc. Example: font-size: 16px;
  • Relative Lengths: Units relative to the parent element’s font size, such as em and rem. Example: font-size: 1.5em; (1.5 times the parent element’s font size)
  • Percentages: Relative to the parent element’s font size. Example: font-size: 150%; (150% of the parent element’s font size)
  • Keywords: Predefined values like medium, large, small, etc. Example: font-size: larger;

These values allow flexibility in specifying font sizes, catering to both absolute and relative measurements

Combining Selectors

CSS Combinators: A combinator is something that explains the relationship between the selectors.

A CSS selector can contain more than one simple selector. Between the simple selectors, we can include a combinator.

There are four different combinators in CSS:

  • descendant selector (space)
  • child selector (>)
  • adjacent sibling selector (+)
  • general sibling selector (~)
